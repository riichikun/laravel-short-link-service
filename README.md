#  Сервис коротких ссылок
Система предоставляет API для генерации коротких ссылок, редиректа по ним и получения статистики переходов по этим ссылкам

## Запуск приложения

Перед запуском приложения нужно создать файл окружения "./src/.env". Это можно сделать путем копирования .env.example:
`cp ./src/.env.example ./src/.env`

В новом созданном файле вводим нужные данные для доступа к БД (в данном образе по умолчанию используется PostgreSQL). Обязательно назначаем пароль.

Поднимаем docker-контейнер с помощью команды:
`docker compose --env-file ./src/.env up -d --build`


**Важно!** Переменные окружения, заданные в .env в директории проекта Laravel, привязаны к данным для доступа к БД. Для их корректного чтения в docker-контейнере необходимо использовать флаг "--env-file" при запуске контейнера

Далее устанавливаем все нужные зависимости composer:
`docker compose exec app composer install`

**Важно!** При возникновении проблем с правами доступа нужно убедиться, что у пользователя, от лица которого запускается docker, есть доступ к директории ./src

Вызываем миграции в контейнере с помощью:
`docker compose exec app php artisan migrate`

Генерируем ключ приложения:
`docker compose exec app php artisan key:generate`


Приложение будет доступно по адресу:
> http://localhost:8000/


Автоматические тесты можно запустить с помощью команды:
`docker compose exec app php artisan test --group=Feature`
